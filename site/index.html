Features

What the dashboard proves

Fleet tracking

GPS pings become trips with distance, duration, speed, and utilization metrics.

Shipment SLAs

Hub scans become shipment journeys with SLA status, hop count, and outcomes.

Last-mile ops

Delivery events become agent, zone, success-rate, and customer-rating views.

Engineering

How it is built

Data pipeline

Simulators create events. Batch jobs reconstruct trips, journeys, and shifts. dbt builds marts in DuckDB.

Quality controls

Schema, freshness, and business-rule checks write JSON artifacts consumed by the dashboard.

Warehouse model

Fact tables cover trips, shipments, hub daily metrics, agent daily metrics, and zone daily metrics.

Runnable demo path

The sample path runs locally without Kafka or Airflow: parquet to DuckDB to dbt to Streamlit.

Flow

  1. Generatefleet, shipment, delivery events
  2. Reconstructtrips, journeys, agent shifts
  3. Modeldbt marts in DuckDB
  4. Validatequality reports and dbt tests
  5. ServeStreamlit dashboard
